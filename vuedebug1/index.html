Vue.js 2.0 除錯紀錄

按下表單 button 送出後 沒有任何反應

在 f12 chrome deveopler，是在 callAPI 之前。在此設下斷點，初步判斷是送出後在 inputdata()這個函式，裡面有一個物件寫錯了，應該更正成 ntfinfodata，這是前端用來放表單資訊的物件，就解決了這個卡 3 小時的問題 = =

前端傳資料給後端 現在發現 user 欄位為空值

原因，在 userInfo 時，沒有在 created 就放入這個函式，導致頁面初始化時取得道的 empNo 為空值 解決方法

Vue.js Ntf Maintain 如何 save 資料 並傳給後端? 按下 save 案件->bindinputData 合併要傳給後端的資料(GetUserInfo 去取得 UserSno BqaInfoForm 去取得子 component 傳值的資料、修改日期格式)成物件-> return 回你要傳給後端的資料->axios 去 call API 去傳資料給後端

如何除此 bug? TypeError: Cannot read properties of null (reading ‘ciphertext’)

[Vue warn]: Error in render: “TypeError: Cannot read properties of null (reading ‘ciphertext’)”

因為是 ciphertext 是空值，代表這是 aes 加密過的文字變成空值。所以要先從登入登入成功後。才能正確顯示 menu 跟 header。
